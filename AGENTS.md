# Techem Project Rules

## Project Memory

Always update both local memory files when something important changes:

- `PROJECT_PLAN.md`
- `DEVELOPMENT_NOTES.md`

This includes:

- new architecture decisions
- changes in migration strategy
- repository structure changes
- validation outcomes
- important implementation milestones
- important reversals or rejected approaches

Expected usage:

- `PROJECT_PLAN.md` tracks status, completed work, next work, and beta acceptance.
- `DEVELOPMENT_NOTES.md` tracks reasoning, tradeoffs, and long-term memory for future work.

These files are local working memory and are intentionally ignored by git, but they must still be kept current.
